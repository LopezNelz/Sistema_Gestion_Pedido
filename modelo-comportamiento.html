Modelo de Comportamiento

Introducción DFD Nivel 1

El Modelo de Comportamiento de Nivel 1, también conocido como Diagrama de Flujo de Datos (DFD) de contexto detallado, representa la descomposición inicial del sistema único identificado en el Modelo Ambiental. Este diagrama proporciona una vista interna de los componentes funcionales principales que conforman el "Sistema de Gestión de Pedidos de Pollos Yack's".

Descripcion del DFD Nivel 1

El DFD Nivel 1 toma el círculo central del Modelo Ambiental y lo reemplaza con una representación detallada de sus procesos internos, manteniendo las mismas entidades externas.

Componentes Principales:

1. Procesos Principales (4 procesos identificados):

  • P1: Gestionar Pedidos de Clientes - Coordina toda la interacción con los clientes
  • P2: Administrar Abastecimiento y Stock - Maneja la relación con proveedores y control de inventario
  • P3: Coordinar Logística de Entrega - Gestiona el proceso de delivery
  • P4: Reportar y Apoyar a la Gerencia - Procesa información para la toma de decisiones

2. Almacenes de Datos:

  • A1: Base de Datos de Pedidos - Almacena todos los pedidos realizados
  • A2: Inventario de Productos - Registra el stock disponible de pollos y productos relacionados
  • A3: Archivo de Clientes - Contiene información de clientes registrados
  • A4: Registro de Proveedores - Mantiene datos de proveedores y transacciones
  • A5: Base de Datos de Entregas - Almacena información de logística y delivery

3. Flujos de Datos Principales:

  • Entre procesos y entidades externas
  • Entre procesos y almacenes de datos
  • Entre diferentes procesos del sistema

Introducción DFD Nivel 2

El Modelo de Comportamiento de Nivel 2 representa la descomposición detallada de cada proceso principal identificado en el Nivel 1. Este nivel proporciona una visión más granular de las funcionalidades específicas dentro de cada módulo del sistema.

Descripcion del DFD Nivel 2 por Proceso

P1: GESTIONAR PEDIDOS DE CLIENTES

Subprocesos:

  • P1.1: Recepción de Pedidos - Captura y validación inicial de solicitudes
  • P1.2: Verificación de Disponibilidad - Consulta de inventario
  • P1.3: Confirmación de Pedido - Generación de confirmación y detalles
  • P1.4: Actualización de Estado - Seguimiento del ciclo de vida del pedido

Flujos Especificos:

  • Cliente → P1.1: Solicitud de pedido
  • P1.2 → A2: Consulta de disponibilidad
  • P1.3 → Cliente: Confirmación y detalles
  • P1.4 → A1: Registro de estado del pedido
  • P1.4 → P3: Activación de proceso de entrega

P2: ADMINISTRAR ABASTECIMIENTO Y STOCK

Subprocesos:

  • P2.1: Recepción de Facturas - Procesamiento de documentos de proveedores
  • P2.2: Control de Inventario - Monitoreo de niveles de stock
  • P2.3: Generación de Órdenes de Compra - Creación de pedidos a proveedores
  • P2.4: Procesamiento de Pagos - Gestión de transacciones financieras

Flujos Especificos:

  • Proveedores → P2.1: Envío de facturas
  • P2.2 → A2: Actualización de niveles de inventario
  • P2.3 → Proveedores: Envío de pedidos procesados
  • P2.4 → A4: Registro de transacciones de pago

P3: COORDINAR LOGÍSTICA DE ENTREGA

Subprocesos:

  • P3.1: Asignación de Delivery - Distribución de pedidos a repartidores
  • P3.2: Seguimiento de Entregas - Monitoreo en tiempo real
  • P3.3: Confirmación de Entrega - Validación de entrega completada
  • P3.4: Manejo de Incidencias - Gestión de problemas en la entrega

Flujos Especificos:

  • P1 → P3.1: Transferencia de pedido para entrega
  • Delivery → P3.2: Estados de entrega
  • P3.3 → Cliente: Confirmación de entrega
  • P3.4 → P1: Notificación de problemas

P4: REPORTAR Y APOYAR A LA GERENCIA

Subprocesos:

  • P4.1: Consolidación de Datos - Agregación de información operativa
  • P4.2: Generación de Reportes - Creación de informes estructurados
  • P4.3: Análisis de Indicadores - Procesamiento de métricas clave
  • P4.4: Soporte a Decisiones - Proporcionar información para decisiones

Flujos Especificos:

  • A1, A2, A5 → P4.1: Extracción de datos operativos
  • Gerencia → P4.4: Parámetros y decisiones
  • P4.2 → Gerencia: Reportes de ventas y rendimiento
  • P4.3 → A5: Almacenamiento de análisis históricos
