Diagrama de Casos de Uso de Negocio

Proposito:

Representar las interacciones entre los distintos roles del negocio y los procesos comerciales principales de "Pollos Jack", independientemente de la tecnología o sistemas específicos.

Elementos Clave:

  • Actores de Negocio: Roles organizacionales (Cliente, Recepcionista, Repartidor, Proveedor, Gerente).
  • Casos de Uso de Negocio: Procesos comerciales fundamentales como "Gestionar Pedido", "Procesar Reservas", "Controlar Inventario", "Generar Reportes".
  • Relaciones: Colaboraciones entre roles para cumplir objetivos del negocio

Alcance:

Se enfoca en qué hace el negocio, no en cómo lo hace tecnológicamente. Muestra las operaciones comerciales desde una perspectiva organizacional, identificando quién participa en cada proceso y qué valor aporta cada interacción.

Diagrama de Casos de Uso de Sistema

Proposito:

Especificar las funcionalidades que el "Sistema de Gestión de Pedidos de Pollos Jack" debe proveer a sus usuarios, definiendo claramente las interacciones entre los actores y el sistema.

Elementos Clave:

  • Actores del Sistema: Usuarios directos del sistema (Cliente, Recepcionista, Repartidor, Administrador)
  • Casos de Uso del Sistema: Funcionalidades específicas del software como "Realizar Pedido Online", "Confirmar Entrega", "Registrar Reserva", "Generar Reporte de Ventas"
  • Relaciones: Inclusión, extensión y generalización entre casos de uso

Alcance:

Define cómo el sistema informático soporta las operaciones del negocio. Especifica los requisitos funcionales desde la perspectiva del usuario, mostrando qué puede hacer cada actor con el sistema y bajo qué condiciones.
